# 状态指示器优化

## 问题

原有的状态指示器逻辑过于简单，只有三个状态：
1. `'工具执行中...'` - 有工具在运行时
2. `'处理结果中...'` - 用过工具但没有在运行时（**模糊不清**）
3. `'思考中...'` - 没有用过工具时

**核心问题：** "处理结果中..." 这个状态太笼统，用户无法准确了解当前处于哪个阶段：
- 工具执行完正在分析结果？
- 正在思考下一步？
- 正在生成最终回答？

## 解决方案

基于 `timeline` 的最后一项内容，提供更精确的状态指示：

### 新的状态逻辑

```typescript
const getStatusText = () => {
  if (hasRunningTool) return '工具执行中...';
  if (lastWasThought) return '思考中...';
  if (timeline.length === 0) return '思考中...';
  // 工具执行完毕后，等待下一阶段（生成回答或继续思考）
  return '分析结果中...';
};
```

### 状态转换流程

1. **初始状态**（timeline 为空）→ "思考中..."
2. **正在思考**（timeline 最后是 thought）→ "思考中..."
3. **工具执行中**（有工具在运行）→ "工具执行中..."
4. **工具执行完毕**（timeline 最后是 tool，工具已完成）→ "分析结果中..."
5. **生成回答**（有文本在流式输出）→ 不显示状态，直接显示文字+光标

### 改进点

- ✅ **更精确**：用户能清楚知道当前处于哪个阶段
- ✅ **更自然**：工具完成后显示"分析结果中..."而不是模糊的"处理结果中..."
- ✅ **更智能**：根据 timeline 内容自动判断状态，无需额外的 `hasUsedTools` 状态

## 代码变更

### 修改的文件

- `frontend/src/App.tsx`

### 主要改动

1. **移除 `hasUsedTools` 状态**
   - 删除了 `useState` 声明
   - 删除了所有 `setHasUsedTools` 调用
   - 从 `LiveAgentRow` 组件的 props 中移除

2. **优化状态判断逻辑**
   - 基于 `timeline` 最后一项判断当前阶段
   - 新增 `lastWasThought` 检查
   - 实现 `getStatusText()` 函数

3. **简化组件接口**
   - `LiveAgentRow` 不再需要 `hasUsedTools` prop
   - 所有状态信息都可以从 `timeline` 中推导

## 用户体验提升

用户现在可以看到更明确的状态提示：
- 开始对话时："思考中..."
- 调用工具时："工具执行中..."
- 工具完成后："分析结果中..."
- 继续思考时："思考中..."

这让整个交互过程更加透明和可预测。
